## Задача

1. Написать Unit на автозапуск Node Exporter
   - Разобораться с основными директивами юнита
   - Поставить бинарь на автозапуск
   - Прописать путь к бинарю
   - Разобраться куда кладут (директория запуска) кастомные юниты в систему
   - Тоже самое с .bin файлами (какая основная директория хранения кастомных бинарей)
   - Разобраться с systemd и как он связан с юнит файлом
   - Не забыть от каких прав запускать бинарь и какие права выдавать юниту
       - Если бинарь prometheus значит и пользователь должен быть prometheus в юните и тд 
2. Написать Unit на автозапуск Prometheus
   - Выше всё было подробно оговорено 
3. Все сервисы должны автоматически стартовать после перезапуска Linux (подумать как это сделать с контейнером Grafana)
4. Развернуть Docker, дать ему необходимые права, добавить в правильную группу пользователей
    - Бегло ознакомиться с основными командами Docker и зачем нужны и вообще как запускать и стопать контейнеры:
        - docker `run -d/-it/p`
        - docker `stop name_id_container`
        - docker `ps`
        - docker `ps -a`
        - docker `logs name_id_container`
        - docker `rm/rmi`
5. Развернуть контейнер Grafana, пользоваться Hub Docker
6. Скачать 3 дашборда (Prometheus, Grafana, Node Exporter) формата Json с официального сайта Grafana Labs и подложить в Grafana - Json именно скачать, айдишник не копируем
7. Подумать, как в контейнер пробросить 3 Json файла на дашборды (использовать ИИ)
8. Все дашборды должны запускаться и отображать все метрики, для этого нужно обратиться к ИИ и подумать с ним вместе каким образом настроить переменные в Grafana и PromQL - чтобы рисовалась вся информация на всех трех дашбордах
9. Задания разложены по папкам, работай с этими папками
10. Сделай форк этого репозитория и склонируй к себе на Ubuntu
11. После завершения - отчет
12. По завершению всех заданий должен быть рабочий мониторинг из трех дашбордов с рабочим автозапуском, дашборды должны показывать всю информацию, а не 30/70% данных, особенно это касается Node Exporter
