## Задача

1. Написать Unit на автозапуск Node Exporter
   - Разобораться с основными директивами юнита
   - Поставить бинарь на автозапуск
   - Прописать путь к бинарю
   - Разобраться куда кладут (директория запуска) кастомные юниты в систему
   - Тоже самое с `.bin` файлами (какая основная директория хранения кастомных бинарей)
   - Разобраться с systemd и как он связан с юнит файлом
   - Не забыть от каких прав запускать бинарь и какие права выдавать юниту
       - Если бинарь prometheus значит и пользователь должен быть prometheus в юните и тд 
2. Написать Unit на автозапуск Prometheus
   - Выше всё было подробно оговорено 
3. Все сервисы должны автоматически стартовать после перезапуска Linux (подумать как это сделать с контейнером Grafana)
4. Развернуть Docker, дать ему необходимые права, добавить в правильную группу пользователей
    - Бегло ознакомиться с основными командами Docker и зачем нужны и вообще как запускать и стопать контейнеры:
        - docker `run -d/-it/p`
        - docker `stop name_id_container`
        - docker `ps`
        - docker `ps -a`
        - docker `logs name_id_container`
        - docker `rm/rmi`
5. Развернуть контейнер Grafana, для этого воспользоваться Hub Docker
6. Скачать 3 дашборда (Prometheus, Grafana, Node Exporter) формата `.json` с официального сайта Grafana Labs и подложить их в контейнер Grafana - `.json` именно скачать, айдишник не копируем
7. Подумать, как в контейнер пробросить 3 `.json` файла на дашборды (использовать ИИ для помощи)
8. Все дашборды должны запускаться и отображать все метрики во всех виджетах, для этого нужно обратиться к ИИ и подумать с ним вместе каким образом настроить переменные в Grafana и PromQL - чтобы рисовалась вся информация на всех трех дашбордах
9. Подумать, как сделать 3 в 1 дашборд: Grafana + Prometheus + Node Exporter, то есть один такой больщой дашборд, при этому внутри страницы он разделяется отдельными заголовками: Grafana + Prometheus + Node Exporter
10. Задания разложены по папкам, работай с этими папками
11. Сделай форк этого репозитория и склонируй к себе на Ubuntu
12. По завершению всех заданий - должен быть рабочий мониторинг из трех дашбордов с рабочим автозапуском (хотя бы на Prometheus и Node Exporter), дашборды должны показывать всю информацию, а не 30/70% данных, особенно это касается Node Exporter
    - Тут подсказка
       - Смотреть в переменные Grafana
       - Смотреть конфиг Prometheus и задавать правильные лэйблы
       - Прописывать правильные лэйблы в самой Grafana
       - Здесь про Prometheus мои статьи, читаем на данном этапе [с 1 по 7](https://github.com/lamjob1993/linux-monitoring/tree/main/tasks/prometheus/beginning)
       - Про настройку [Grafana](https://github.com/lamjob1993/linux-monitoring/tree/main/tasks/grafana)
       - Что мониторит [Node Exporter](https://github.com/lamjob1993/linux-monitoring/tree/main/tasks/node-exporter) и зачем
